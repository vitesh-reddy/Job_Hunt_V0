# Job Hunt Backend

A Node.js/Express backend for the Job Hunt application, providing RESTful APIs for user authentication, job tracking, resume management, cover letters, and templates.

## Features

- User registration, login, OTP verification, and password reset
- User profile management
- Job tracking and status updates
- Resume and cover letter management
- Template management
- JWT-based authentication
- Email and SMS notifications

---

## Tech Stack

- **Node.js** (JavaScript runtime)
- **Express.js** (Web framework)
- **MongoDB** (Database, via Mongoose)
- **Nodemailer** (Email service)
- **JWT** (Authentication)
- **dotenv** (Environment variables)

## Folder Structure

```
backend/
│
├── config/         # Configuration files (DB, logger, email, etc.)
├── controllers/    # Route handler logic for each resource (auth, user, jobs, etc.)
├── logs/           # Log files generated by the application
├── middlewares/    # Express middlewares (auth, logging, etc.)
├── models/         # Mongoose schemas and models for MongoDB
├── routes/         # API route definitions for each resource
├── services/       # Business logic and helper services (email, user, job, etc.)
├── uploads/        # Uploaded files (e.g., resumes, profile pictures)
├── utils/          # Utility functions (JWT, OTP, email, SMS helpers, etc.)
│
├── .env            # Environment variables for backend config
├── .gitignore      # Files and folders to ignore in git
├── API_Documentation.md # API endpoint documentation
├── package.json    # Project metadata and dependencies
├── README.md       # Project documentation
└── server.js       # Entry point for the Express server
```

## Getting Started

1. **Clone the repository:**
   ```sh
   git clone https://github.com/Variantlabs1/Project--js.git
   cd backend
   ```

2. **Install dependencies:**
   ```sh
   npm install
   ```

3. **Set up environment variables:**
   - Copy `.env.example` to `.env` and fill in your values.

4. **Start the server:**
   ```sh
   npm start
   ```
   The server will run on the port specified in your `.env` file.

---

## Environment Variables

Create a `.env` file in the root directory. Example variables:

```
PORT=5000
MONGO_URI=mongodb://localhost:27017/job_hunt
JWT_SECRET=your_jwt_secret
EMAIL_USER=your_email@example.com
EMAIL_PASS=your_email_password
```
